# **在庫管理スキャナー 反映手順マニュアル**

修正したコードを「GitHub（フロントエンド）」と「GAS（バックエンド）」の両方に正しく反映させ、スマホアプリを更新するための手順書です。

## **1\. 事前確認（設定ファイルのチェック）**

以下のファイルが正しく設定されているか確認してください。

### **.claspignore**

GASに不要なファイルを送らないための設定です。

\*\*/\*\*  
\!script.js  
\!appsscript.json

### **sw.js (キャッシュ更新)**

スマホ側に「新しいプログラムがあるよ！」と知らせるために、バージョン番号を必ず1つ上げます。

const CACHE\_NAME \= 'inventory-v1.0.3'; // 数字をカウントアップ

## **2\. GitHubへの反映（フロントエンド）**

index.html や sw.js などの画面側の修正を公開します。

1. ターミナルでプロジェクトフォルダに移動  
2. git add .  
3. git commit \-m "修正内容のメモ（例：在庫表示のバグ修正）"  
4. git push

## **3\. GASへの反映（バックエンド）**

script.js のロジックをGoogleサーバーに送信します。

1. ターミナルで clasp push を実行  
2. **ブラウザで [GASエディタ](https://script.google.com/) を開く**  
3. 左メニューから sw.gs や index.html が混入していないか確認（あれば削除）  
4. 右上の **「デプロイ」** ＞ **「デプロイを管理」** をクリック  
5. 現在のデプロイの **「鉛筆アイコン（編集）」** をクリック  
6. バージョンを **「新バージョン」** に変更  
7. **「デプロイ」** ボタンを押して完了

## **4\. スマホアプリの更新確認**

1. スマホのホーム画面にあるアイコン（PWA）を一度完全に閉じる  
2. アプリを起動する  
3. 画面上部に表示されるバージョン（Ver: V1.0.3など）が更新されているか確認  
   * 更新されない場合は、画面を下にスワイプしてリロードしてください。

## **トラブルシューティング**

* **「送信失敗」が出る**：GASのURLが最新のデプロイURLと一致しているか確認してください。  
* **画面が変わらない**：sw.js の CACHE\_NAME を変え忘れている可能性があります。  
* **GASでエラーが出る**：GASエディタ内に sw.gs などの不要なファイルが残っていないか確認してください。
