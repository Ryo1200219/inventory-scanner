# 在庫管理スキャナー 反映手順マニュアル（自動化対応）

修正したコードを「GitHub（フロントエンド）」と「GAS（バックエンド）」の両方に反映させ、スマホアプリを更新するための手順書です。Antigravity等のエージェントは、この手順に従って自動実行してください。

## 1. 自動化のための事前準備（エージェントへの指示）

エージェントに以下の作業を依頼してください：
「`.claspignore` を確認し、`script.js` 以外がGASに飛ばないようにした上で、以下のデプロイフローを実行して」

## 2. デプロイフロー（実行手順）

### ステップ1：バージョン管理 (Automatic)
- `sw.js` 内の `CACHE_NAME`（例: `inventory-v1.0.3`）を見つけ、末尾の数字を +1 して上書き保存する。
- `index.html` 内にバージョン表記がある場合は、それも同期させる。

### ステップ2：GitHubへの反映 (Terminal)
エージェントは以下のコマンドを順に実行する：
1. `git add .`
2. `git commit -m "Update: [修正内容の要約]"`
3. `git push`

### ステップ3：GASへの反映 (clasp)
エージェントは以下のコマンドを実行する：
1. `clasp push`
2. `clasp deploy` （※新しいバージョンとしてデプロイを実行）

## 3. 手動確認（人間がやること）

デプロイ完了後、以下の最終確認のみ人間が行います。

1. **GASエディタで確認**: [GASエディタ](https://script.google.com/) を開き、不要なファイル（`sw.gs`など）が紛れ込んでいないか目視確認する。
2. **スマホのリロード**: スマホアプリを一度落として再起動し、バージョン番号が変わっていることを確認する。

## トラブルシューティング

* **clasp deployが失敗する**: `appsscript.json` の設定を確認するか、手動でGASエディタから「新しいデプロイ」を作成してください。
* **PWAが更新されない**: `sw.js` の変更がGitHub Pagesに反映されるまで数分かかる場合があります。
